🎭 Алиби — Система онлайн-бронирования квестов
Курсовой проект по МДК.01.01 «Разработка программных модулей»
Веб-приложение для автоматизации бизнес-процессов квест-рума: от управления каталогом и расписанием до взаимодействия с клиентами и генерации отчетности.
________________________________________
🚀 Ключевые возможности
Для клиентов:
•	Интерактивный каталог: поиск квестов с фильтрацией по жанрам, сложности, уровню страха и количеству игроков.
•	Умное бронирование: выбор даты и времени с проверкой доступности слотов (защита от «двойного бронирования»).
•	Личный кабинет:
o	История бронирований и отслеживание статусов.
o	Внесение виртуальной предоплаты.
o	Генерация и скачивание электронных чеков в формате .txt.
o	Возможность оставить отзыв и оценку только после прохождения игры.
•	Уведомления: автоматическая рассылка подтверждений и отмен на Email.
Для администраторов:
•	Dashboard: наглядная статистика по выручке, количеству квестов и бронирований.
•	Управление каталогом (CRUD): добавление, редактирование и удаление квестов (с загрузкой изображений).
•	Контроль заявок: просмотр всех бронирований с возможностью принудительной отмены.
•	Генерация отчетов: выгрузка детальной аналитики по квестам и бронированиям в формате MS Word (.docx) с логотипом и печатью организации.
________________________________________
🛠 Технологический стек
•	Backend: Python 3.10+, FastAPI (Asynchronous Framework)
•	Database: PostgreSQL + SQLAlchemy (ORM)
•	Frontend: HTML5, CSS3 (Custom Grid/Flexbox), JavaScript (Vanilla, Fetch API)
•	Reporting: python-docx (генерация документов Word)
•	Authentication: Starlette Sessions, Custom Auth Logic
•	Email: SMTP (интеграция с Mail.ru)
•	Testing: Pytest

📦 Установка и запуск
1. Подготовка окружения
Убедитесь, что у вас установлен Python 3.10+ и PostgreSQL.
codeBash
# Клонировать репозиторий
git clone https://github.com/твой-логин/alibi-quest-site.git
cd alibi-quest-site

# Создать и активировать виртуальное окружение
python -m venv venv
source venv/bin/activate  # Для Linux/macOS
venv\Scripts\activate     # Для Windows
2. Установка зависимостей
codeBash
pip install -r requirements.txt
(Если файла requirements.txt нет, установи основные библиотеки: pip install fastapi uvicorn sqlalchemy psycopg2-binary jinja2 python-docx python-multipart python-jose)
3. Настройка базы данных
1.	Создайте базу данных в PostgreSQL с названием Quest_site_kur.
2.	В файле database.py при необходимости измените данные для подключения (пользователь, пароль).
4. Запуск сервера
codeBash
python main.py
# ИЛИ
uvicorn main:app --port 5000 --reload
Приложение будет доступно по адресу: http://127.0.0.1:5000

📂 Структура проекта
•	main.py — точка входа, маршрутизация и бизнес-логика.
•	models.py — описание моделей базы данных SQLAlchemy.
•	crud.py — функции для работы с данными (Create, Read, Update, Delete).
•	auth.py — логика авторизации и проверки прав доступа.
•	review_utils.py — вспомогательные функции для работы с рейтингом и отзывами.
•	email_utils.py — модуль отправки почтовых уведомлений.
•	templates/ — HTML-шаблоны Jinja2.
•	static/ — CSS-стили, клиентский JavaScript и медиа-файлы.
•	test_main.py — автоматизированные тесты функционала.

📝 Особенности реализации
1.	Бизнес-правило 24 часов: система блокирует возможность бронирования и отмены квеста, если до сеанса осталось менее суток.
2.	Безопасность: доступ к админ-панели ограничен на уровне декораторов (проверка флага is_admin).
3.	Оптимизация БД: используются хранимые процедуры PostgreSQL для быстрого поиска и фильтрации квестов.
